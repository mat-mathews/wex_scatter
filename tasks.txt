CI/CD CHECK MODE (HIGH PRIORITY)
- [ ] Machine-readable output with exit codes for pipeline integration
      Exit 0 = no concerns, Exit 1 = warnings, Exit 2 = high-risk changes
- [ ] --fail-on-high-risk flag: non-zero exit if blast radius > N projects
- [ ] --output-format json for structured consumption by pipeline scripts
- [ ] Configurable thresholds via .scatter.yaml

PR BLAST RADIUS COMMENTS (HIGH PRIORITY)
- [ ] Post impact analysis as PR comment (GitHub)
- [ ] Post impact analysis as PR comment (Azure DevOps)
- [ ] Include: affected projects, risk level, pipelines to test
- [ ] Configurable: suppress comment if blast radius = 0
- [ ] Provide GitHub Action template
- [ ] Provide Azure DevOps pipeline task template

AUTOMATIC BASELINE UPDATES (MEDIUM PRIORITY)
- [ ] Update baseline automatically on merge to main
- [ ] Store baseline history (keep last N baselines)
- [ ] Trigger via CI/CD pipeline or git hook

COUPLING THRESHOLDS (MEDIUM PRIORITY)
- [ ] Configurable gates in .scatter.yaml:
      max_blast_radius: 10 (warn if more than N projects affected)
      max_coupling_increase: 5% (warn if coupling score increases)
      block_new_cycles: true (fail if PR introduces circular dependency)
- [ ] Output threshold violations in CI/CD check mode

WATCH MODE (LOW PRIORITY)
- [ ] `scatter analyze --watch` — re-run analysis on file changes
- [ ] Use filesystem events (watchdog or similar)
- [ ] Useful for interactive local exploration during development
- [ ] Debounce rapid changes (wait 500ms after last change)

PROGRESS BAR UI (LOW PRIORITY)
- [ ] Add tqdm progress bars for long-running scans
- [ ] Show progress per phase: file discovery, project mapping, consumer analysis
- [ ] Suppress in --quiet mode and CI/CD check mode

PHASE 5 TESTING
- [ ] Unit tests for exit code logic (threshold evaluation)
- [ ] Unit tests for PR comment formatting (GitHub + Azure DevOps)
- [ ] Integration test: CI/CD check mode with known blast radius
- [ ] Integration test: threshold violations trigger correct exit codes
- [ ] Integration test: watch mode detects file changes and re-runs
